# دليل الإعداد الأولي — منصة حلّل

دليل مختصر للمدير الأول بعد تثبيت المنصة.

## 1. تغيير كلمة المرور

1. سجّل الدخول بحساب المدير (`0500000000` أو كلمة المرور من `ADMIN_INITIAL_PASSWORD`).
2. سيُطلب منك تغيير كلمة المرور عند أول دخول — اختر كلمة مرور قوية واحفظها.

## 2. إنشاء الأقسام والهيكل التنظيمي

1. من **الأقسام** أنشئ الأقسام (مثل: الإدارة التنفيذية، المشاريع، المالية).
2. من **الفريق** أضف المدراء واربط كل مدير بقسمه.
3. عيّن `manager_id` لكل موظف لبناء الهيكل الإداري.

## 3. إضافة العاملين

1. من **الفريق** → **إضافة مستخدم**.
2. أدخل الاسم، الجوال، القسم، والمدير المباشر.
3. فعّل الحساب (`is_active`) واطلب تغيير كلمة المرور عند أول دخول.

## 4. إسناد الأدوار الخمسة

| الدور (إنجليزي) | التسمية العربية | الاستخدام |
|-----------------|-----------------|-----------|
| General Manager | المدير العام | عرض شامل + اعتماد ودفع المصروفات + إدارة العقود |
| Executive Manager | المدير التنفيذي | المشاريع، المهام، الاجتماعات، التقارير |
| Project Manager | مدير مشروع | مشروعه ومهامه واجتماعاته ومصروفاته |
| Finance | المالية | الرواتب، المصروفات، العقود، التقارير |
| Employee | موظف | مهامه، اجتماعاته، مستنداته، طلبات المصروف |

من **الأدوار والصلاحيات** أو عند إنشاء المستخدم عيّن الدور المناسب. لا تمنح **مدير النظام (Super Admin)** إلا لمسؤول تقني واحد.

## 5. (اختياري) بيانات تجريبية

لتجربة الأدوار بسرعة:

```bash
php artisan db:seed --class=OnboardingSeeder
```

ينشئ 3 أقسام و5 مستخدمين تجريبيين (`0501111111`–`0505555555`) بكلمة مرور من `ADMIN_INITIAL_PASSWORD` أو `OnboardDemo2026!`.

## ترتيب البذر الكامل

```bash
php artisan migrate:fresh --seed
php artisan db:seed --class=OnboardingSeeder   # اختياري
```

## ملاحظات أمنية

- الصلاحيات تُفرض من الخادم فقط — لا تعتمد على إخفاء روابط الشريط الجانبي.
- كل دور يحصل على أقل صلاحيات ممكنة (least privilege).
- الموظف يرى المصروفات لأنه يحتاج `expenses.view` لمتابعة طلباته.
