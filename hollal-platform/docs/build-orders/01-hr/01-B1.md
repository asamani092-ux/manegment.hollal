## Objective

Employee directory + job profile page (tabs, field-level permissions, profile access logging).

## Spec reference

docs/specs/spec-01-hr.md §1–§2

## Deliverables

- Employee cards (+ optional wide table)
- Job profile page with internal tabs
- Field-level permissions (e.g. salary tab)
- profile_access_logs on sensitive tab opens

## Required grep proofs

- `hr.employees.view|hr.salaries.view`
- `profile_access_logs`
- `ds-task-card or employee card classes in HR index`

## Tests

- Employee without hr.salaries.view cannot see salary tab
- Opening salary tab writes access log
- Soft-delete only — no hard delete
