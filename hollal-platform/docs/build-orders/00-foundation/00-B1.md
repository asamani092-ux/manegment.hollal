## Objective

Delete legacy root design-system leftovers, Railway/Render configs (Hostinger decision), unused assets/views — produce deletion list in the commit message; archive nothing needed.

## Spec reference

docs/specs/ (foundation hygiene notes) · Hostinger deploy decision in docs/plans/

## Deliverables

- Removal of unused root design-system leftovers and cloud configs (Railway/Render)
- Cleanup of unused assets/views with explicit deletion list in commit message

## Required grep proofs

- `railway|render.yaml|render.yml — must be absent after cleanup`
- `halal-design-system leftovers paths removed from repo root (as listed in commit)`

## Tests

- Full suite remains green after deletions
- Smoke: routes and assets still load for active screens

## Definition of done (protocol)

1. commit message includes `00-B1`
2. push origin/main + hash range
3. git status clean / up to date
4. `php artisan test` full suite summary pasted
5. grep proofs pasted
6. files created/updated listed
